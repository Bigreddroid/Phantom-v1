1. Vision & Mission

Phantom is the flagship product of BigRedDroid — a solo deep-tech lab building 92 AI-powered products. Phantom is #1.

Phantom automates the entire X/Twitter (and LinkedIn) personal brand for @BigRedDr0id — 24/7, without manual work. It posts content in an authentic voice, engages with the community, replies to mentions, follows relevant accounts, runs educational threads, sends DMs, and grows the audience on autopilot.

  • Platform and proof-of-concept for future BigRedDroid products.

3. X Auth Architecture

No official Twitter API keys are used for posting. Everything goes through browser cookies extracted from Brave.

Cookie chain (in priority order)

  1. X_COOKIES Vercel env var (JSON array of cookie strings) — fastest path
  2. XSession record in Supabase DB (fallback when env var missing)
  3. Fresh login via X_USERNAME + X_PASSWORD — saves to DB for reuse

Two scrapers, one session

  • agent-twitter-client (getXClient()) — used for all write ops: sendTweet, likeTweet, retweet, follow, sendDm
  • @the-convocation/twitter-scraper (getConvoScraper()) — used for search/read ops. Shares the same cookie session from DB. Agent-twitter-client's search returns 404 from Vercel datacenters.

Patches (scripts/patch-agent-twitter.js)

Runs automatically on postinstall and build. Patches agent-twitter-client to:

  • Replace twitter.com → x.com and update API paths
  • Update stale GraphQL query hashes (sourced from @the-convocation@0.22)
  • Add browser headers to write operations (required by X's bot detection)
  • Update bearer token

Cookie refresh procedure

1. Close Brave completely:    Get-Process -Name "brave" | Stop-Process -Force
2. Run cookie extractor:       node get-cookies.js
3. Deploy with new cookies:    npx vercel deploy --prod
   (cookies go into X_COOKIES env var)
⚠️ Cookies expire ~30 days. If posting starts failing with auth errors, run the refresh procedure above.

6. Content System

Tweet deduplication (anti-repetition)

Phantom reads the last 40 posted QueueItems (full untruncated text) and passes them to Claude as a STRICTLY FORBIDDEN list. Claude is instructed to never repeat hooks, phrases, sentence structures, or angles from that list. Both generateTweet and generateBuildUpdate use this.

Reply deduplication (shared pool)

src/lib/reply-dedup.ts — shared library used by all reply crons (mentions, engage, goout, follow). Stores tid:<tweetId>|aid:<authorId> format. Permanent dedup for mentions, 30-day for engage/goout, 7-day author-level dedup. Prevents any tweet from getting two replies from any cron.

Reply tone

Peer-to-peer builder voice. Not a support bot or hype machine. Direct, occasionally dry, genuinely curious. Rules: never start with "I", no hollow openers ("great point", "love this", "so true"), no unsolicited advice, no summarising their tweet back at them. Winning → curious ("what made the difference?"). Struggling → acknowledge briefly + something real ("yeah X always bites — I ran into that doing Y").

Content generation functions (src/lib/claude/generate.ts)

  • generateTweet(topic, recentTweets) — standard tweet, @BigRedDr0id voice
  • generateBuildUpdate(product, context, recentTweets) — "shipped X today" style
  • generateThread(topic, recentTweets) — 5-7 tweet thread
  • generateArticleThread(topic, recentTweets) — 6-tweet educational thread: hook → 3 insight tweets → CTA → summary. Topics from ARTICLE_TOPICS array (26 topics).
  • generateReply(tweetText, authorHandle) — contextual reply, peer-to-peer tone
  • generateGoOutComment(tweetText) — proactive comment, direct + specific
  • generateLinkedInPost(topic) — LinkedIn-specific voice
  • generateDM(handle, context) — personal DM, not salesy
  • generateLongTweet(topic) — Premium+ 600–1800 char post

Article Topics (26 pre-researched)

High-traction categories: Claude AI tips · Notion workflows · Obsidian PKM · AI automation for founders · X/personal brand growth · Solopreneur productivity

Posting flow

  1. Content generated → saved as QueueItem (status: PENDING)
  2. Telegram message sent with Approve / Skip / Regenerate buttons
  3. On approve: postQueueItem() reads meta.imageStyle, calls correct post function
  4. QueueItem status updated to POSTED, Activity record created
  5. Mentions: never auto-approve — manual approval required every time
